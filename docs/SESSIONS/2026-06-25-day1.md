## Decisiones técnicas

- **Vistas como security barrier** (no security_invoker): anon no tiene SELECT a
  la tabla persons; las vistas persons_public/notes_public son el único portal,
  filtran approved+non-withdrawn y enmascaran columnas. security_barrier=true.
- **APP_SALT vía tabla app_config** (RLS deny-all) + get_app_salt() SECURITY
  DEFINER, en vez de GUC de base (Supabase no lo permite con rol postgres).
- **Conexión directa IPv6** (`db.<ref>.supabase.co:5432`) — el Mac tiene IPv6
  ruteable, así que no hace falta el pooler.
- **Runner de migraciones propio** (`scripts/apply-migrations.mjs`, pg) idempotente
  con tracking en `_faro_migrations` — más robusto que el CLI para este caso.
